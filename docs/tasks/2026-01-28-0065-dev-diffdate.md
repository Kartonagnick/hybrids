#65-dev-diffdate
================
Заказ от проекта `stash`  
В ходе разработки то и дело возникает потребность  
в определении длительности периода между датами  

Нужно офф-лайн решение под Windows, которое не требует ни интернета,  
ни какой либо специальной установки.  
И которое теоретически можно кастомизировать под ключ конкретных задач.  

Гибрид `diffdate` идеально подходит для решения такой задачи.  
  - отвечает всем заявленным требованиям  
  - легко реализуется на базе `hybrids`  

На входе:  
 - `eDEBUG=ON`  
 - `eBEG_DATE=2026-01-28 10:00:00`  
 - `eEND_DATE=2026-01-28 12:01:02`  
 - `eALLOW_NEGATIVE=ON`  

Если не было заданно `eBEG_DATE`, то по умолчанию используется текущая дата/время  
Если не было заданно `eEND_DATE`, то по умолчанию используется текущая дата/время  

В нормальном случае `eBEG_DATE` < `eEND_DATE`,  
и тогда гибрид вернет длительность между конечной и начальной датами.  

Если `eBEG_DATE` == `eEND_DATE`, тогда длительность будет 0 секунд  
Если `eBEG_DATE` > `eEND_DATE`, тогда длительность получится отрицательной  

Если длительность получится отрицательной,  
и при этом не был задан режим `eALLOW_NEGATIVE`,  
то гибрид сгенерирует ошибку.
